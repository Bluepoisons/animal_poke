# 甲乙方对辩汇总与 v2 修订建议

> 本文档汇总甲方（发行方/投资人）与乙方（开发团队负责人）两轮交叉质询的成果，提炼共识与遗留问题，作为《开发计划 v2》的修订依据。

---

## 一、对辩概览

| 维度 | 说明 |
|------|------|
| 甲方角色 | 发行方/投资人，关注商业化、获客、合规、ROI、风险 |
| 乙方角色 | 开发团队负责人（主策+技术），关注可行性、设计、技术、体验、排期 |
| 第一轮 | 各自独立审阅开发计划 → 产出质询清单 + 策略建议 + 对方提问 |
| 第二轮 | 交叉投喂对方报告 → 逐条回应 + 反问 → 达成共识 |
| 成果 | 29 项共识、10 项遗留待协商、甲方明确投资意向（签 LOI） |

**对辩价值**：乙方原计划的最大硬伤（商业化空白、永久损伤劝退、合规盲区、排期超载）被甲方精准击中；乙方的技术风险（AI 产能、AR 降级、防作弊漏洞、通胀）也得到正视。双方在 6 项全盘接受、2 项部分接受后达成可执行共识。

---

## 二、核心共识（必须纳入 v2 计划）

### 2.1 商业化架构重构（最大修订，原计划几乎空白）

| 修订点 | 原计划 | v2 共识 |
|--------|--------|---------|
| 货币体系 | 单一金币 | **双货币：金币（软）+ 钻石（硬，1 元 = 10 钻）** |
| 付费入口 | 无 | 月卡 30 元 / 通行证 68 元/赛季 / 首充 6 元 / 礼包 6-648 元梯度 |
| 钻石红线 | — | **不可直接购买 Rare 及以上宠物（上限 Uncommon）；不可购买捕获成功率** |
| 复活药剂 | 商店极低概率刷新 | **改为钻石直购**（80 钻），成为付费转化点 |
| 区域排行奖励 | 全服发金币（通胀源）| **改为钻石 + 称号**，消除通胀 |

**商业化目标**：

| 指标 | 软启动 | 上线 3 月 | 上线 6 月 |
|------|--------|----------|----------|
| 付费率 | 3~5% | 6~8% | 8~12% |
| ARPPU | 25 元 | 35 元 | 45~60 元 |
| ARPU | 0.75~2.0 元 | 2.1~3.2 元 | 3.6~7.2 元 |

### 2.2 机制修复（消除劝退风险）

| 机制 | 原计划 | v2 共识 |
|------|--------|---------|
| 永久损伤 | 感冒自愈后 2~5% 概率永久 -5% | **下线删除** |
| 感冒幅度 | 全属性 -35% 持续 5 天 | **-20% 持续 3 天** |
| 感冒概率 | 雨 20% / 雪 15% | **雨 8% / 雪 6%** |
| 感冒药价格 | 200 金币 | **150 金币** |
| 预防道具 | 无 | **新增"雨衣"（50 金币，雨天出战免疫感冒）** |
| 区域排行 | 省份排行，Top3 发金币 | **城市分区 + 总榜/人均榜/进步榜三榜并行，奖励改钻石** |
| 派遣机制 | 消耗体力 | **改冷却制（1h 冷却）+ 每日次数上限（Lv1=10 / Lv5=15 / Lv10=30）** |
| 稀有度扰动 | ±0.1（可跨 2 级）| **±0.03 且不跨档** |

### 2.3 技术与工程修订

| 修订点 | 原计划 | v2 共识 |
|--------|--------|---------|
| 包体 | ≤400MB | **首包 ≤150MB，模型与资源走动态下发（Unity Addressables）** |
| AI 工程师 | 1 人 | **2 人**（1 训练 + 1 部署）+ 数据标注外包（15 万）+ 算力（12 万）|
| AI 兜底 | 无 | **降级路径**：开源模型微调 / 砍品种 Top 数 / 鹅延后 |
| 检测帧率基线 | ≥10-15 FPS | **≥8 FPS（中端机实测）** |
| AR 降级 | 无 | **三档**：完整 AR → 2D 物理投掷 → GPS 近距离发现 |
| 天气降级 | 单源和风 | **双源（和风+心知）+ 缓存 ≤1h + 跨城强制刷新** |
| 防作弊 | 时间戳+EXIF+轨迹 | **+SafetyNet/DeviceCheck + 帧水印 + Native So + GPS 聚类 + 行为画像** |

### 2.4 排期与团队修订

| 修订点 | 原计划 | v2 共识 |
|--------|--------|---------|
| 总工期 | 8 个月 | **10 个月 + 软启动 1~2 月（上限 12 月）** |
| 内测阶段 | 11 项交付超载 | **砍战斗系统（延后公测），内测延长至 3 月** |
| PvP 排位 | 公测交付 | **延后到上线后版本** |
| 商业化策划 | 公测才招 | **MVP 阶段即引入** |
| 数值策划 | 兼职 | **全职** |
| 鹅物种 | MVP 交付 | **MVP 交付猫+狗，鹅延后至内测** |

**对外宣传口径调整**：原"猫/鹅/狗三物种首发" → **"猫+狗首发，更多物种持续更新"**。

**质量门槛（延期触发条件）**：

| 指标 | 阈值 |
|------|------|
| 端侧检测误检率 | < 8%（中端机实测）|
| 崩溃率 | < 0.5% |
| 中端机帧率 | ≥ 30fps（核心场景）|
| 内测 D1 留存 | ≥ 35% |
| 内测 NPS | ≥ 30（从 40 下调）|

### 2.5 合规分工修订（原计划最大盲区）

| 事项 | 责任方 | 启动节点 |
|------|--------|---------|
| 版号申请 | **甲方主导**，乙方配合 | MVP 第 1 月 |
| 测绘资质 | 用第三方 SDK（高德/百度），不自建 | MVP 第 2 月评估 |
| 未成年防沉迷 | 乙方实现 | 内测阶段 |
| 隐私合规备案 | 甲方法务主导 | MVP 第 2 月 |
| 动物保护 | **黑名单 + 珍稀保护动物不纳入识别** | MVP 阶段 |
| 人脸/车牌模糊 | 乙方端侧实现 | MVP 阶段 |
| 用户协议/隐私政策 | **甲方法务出具** | MVP 阶段 |

**新增风险点**（原计划完全遗漏）：拍摄他人宠物物权、动物保护组织质疑、未成年户外安全、LBS 测绘资质、版号周期 6~12 月。

### 2.6 居家玩法新增（解决"不出门就不玩"）

| 玩法 | 说明 |
|------|------|
| 宠物对战 PvP/PvE | 战斗系统本身即居家核心循环 |
| 宠物训练室 | 每日 3 次，喂食/抚摸/训练，提升经验与心情 |
| 云端图鉴漫游 | 每日 3 次远程观察其他城市图鉴（不捕获），概率获碎片 |
| 好友互访 | 访问好友宠物园，互动获少量金币/碎片 |
| 赛季任务 | 居家可完成，驱动通行证进度 |

**约束**：北方城市（北京/沈阳）冬季专项验证，D7 跌破 15% 则居家玩法必须升级。

### 2.7 数据与分成

- 运营数据归甲方，乙方实时只读看板
- 分成基准以**第三方审计**为准（甲方指定、乙方认可、费用甲方承担）
- 月度对账 + 季度审计 + 年度终审

|------|------|

## 四、遗留待协商问题（10 项，LOI 签订后 2 周内协商）

| 编号 | 遗留问题 | 甲方初步立场 |
|------|---------|------------|
| L1 | 分成比例 | 阶梯：回收前 70/30（甲/乙），回收后 50/50 |
| L2 | IP 归属 | 共同所有，甲方享优先续作权与改编权 |
| L3 | 项目失败资产处置 | 乙方保留代码非独家使用权，甲方留运营数据与渠道 |
| L4 | 竞业限制 | 核心成员合作期及结束后 1 年内不得开发同类产品 |
| L5 | 里程碑验收细则 | 双方各出初稿，2 周合并 |
| L6 | 研发预算拆解 | 乙方 v2 计划提供完整预算表 |
| L7 | 居家玩法冬季验证方案 | 软启动阶段专项设计，双方共定 |
| L8 | 商业化策划岗位 | 乙方招聘、甲方承担薪酬、双线汇报 |
| L9 | 蒙特卡洛经济平衡表验收 | 甲方数值审计顾问参与评审 |
| L10 | 鹅物种延后是否调整内测验收 | 不调整，鹅作为公测交付项 |

## 五、甲方投资意向

**决定：进入下一步，签订《投资意向书》（LOI），启动 v2 计划制定。**

**时间线**：
1. **2 周内**：乙方提交 v2 计划（含完整财务模型、双货币经济系统、蒙特卡洛平衡表、合规时间线、软启动方案）
2. **4 周内**：双方完成 LOI 签订（投资金额、分成、IP、竞业、验收标准）
3. **6 周内**：首笔研发款（30%）到账，项目正式启动

**撤回意向的触发条件**：
- v2 计划仍未提供完整财务模型与商业化设计
- 乙方拒绝竞业限制条款
- 乙方拒绝商业化策划双线汇报
- 乙方单方面将研发预算上浮超 30%

| 原章节 | 修订动作 |
