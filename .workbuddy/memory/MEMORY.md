## 项目约定（重要）
- **设计文档唯一来源**：所有游戏设计改进只写进 `游戏开发计划.md`，不产出额外散落文件（不单独生成对辩汇总、评估图等过程文件）。开发计划.md 是唯一最终方案。
- 用户偏好：先聚焦玩法设计本身，暂不考虑商业化效益。
- **API 密钥管理（v1.3 起）**：**所有第三方 Key（腾讯地图/彩云天气/云端 VLM/云端 LLM）只存 Go 后端的 `.env`，客户端不持有、不接触任何第三方 Key**，客户端仅持后端地址 `BACKEND_BASE_URL` + 设备 Token。`.env` 已加入 .gitignore，禁止硬编码。
- **git 分叉处理**：用 rebase（`git pull --rebase`），保持线性历史。
