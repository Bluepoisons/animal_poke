## F1 修复: GameManager Parse Error (autoload 跨引用)
- 现象: Godot 加载项目报 `game_manager.gd:69 - Parse Error: Identifier "NetworkManager" not declared in the current scope`。
- 原因: game_manager.gd 在 _is_transition_valid 里直接用 autoload 全局名 `NetworkManager.is_offline()`。Godot 在外部修改 project.godot 后若编辑器未重新加载项目, autoload 全局符号未注册到 GDScript 编译器, 导致 Parse Error。
- 修复: 改用运行时节点查找 `get_node_or_null("/root/NetworkManager")` + `nm.call("is_offline")`, 不依赖编译期全局符号。
- 经验(重要): **autoload 脚本之间互引时, 用 `/root/<SingletonName>` 路径运行时查找更健壮**; 普通场景脚本里用全局名无此问题(项目已加载)。外部改 project.godot 后需在编辑器「项目→重新加载当前项目」。
- 任务清单补充: F1 任务下新增「完成记录」小节(交付物/初始化顺序/验收结果/关键决策), 作为完成证据归档。F1 正式收尾。

## 修复 MainScreen _update_layout Nil 错误
- 报错：`Invalid assignment of property or key 'custom_minimum_size' with value of type 'Vector2' on a base object of type 'Nil'`，位置 `main_screen.gd:123`。
