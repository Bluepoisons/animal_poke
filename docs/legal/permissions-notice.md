# Animal Poke 应用权限申请与使用说明

**版本号**：v1.0.0  
**更新日期**：2026-07-10  

根据工业和信息化部等关于 App 违法违规收集使用个人信息的治理要求，我们仅在业务功能需要时申请相关权限，并在调用前告知用途。

| 权限 | 对应系统能力 | 使用目的 | 申请时机 | 拒绝后影响 |
|------|--------------|----------|----------|------------|
| **相机 / 摄像头** | Camera / getUserMedia | 预览与拍摄动物图像，供识别与捕获 | 进入发现 / 捕获并征得隐私同意后 | 无法使用发现与捕获；可浏览图鉴 |
| **位置（精确 / 大致）** | Location / Geolocation | 逆地理、猎取地图、区域与天气相关玩法 | 使用地图 / 需定位的功能时 | 无法展示真实城市与 LBS 内容；可降级 |
| **网络** | Internet | 与服务器通信、AI 识别、同步 | 安装后系统默认或首次联网 | 无法在线游玩（在线优先架构） |
| **存储 / 相册（若后续支持选图）** | Photo Library | 用户主动选择本地图片识别（如启用） | 用户触发选图时 | 仅无法使用选图；拍照仍可能可用 |
| **通知（若启用）** | Notifications | 体力恢复、活动提醒等 | 用户主动开启时 | 无法接收推送，不影响核心玩法 |

## 我们不会申请（MVP 无合理场景时）

- 通讯录、短信、通话记录、日历  
- 麦克风（无语音功能时）  
- 持续后台定位（无必要不开启）  
- 读取已安装应用列表用于与功能无关的画像  

## 权限管理路径（通用说明）

- **iOS**：设置 → 隐私与安全性 → 相机 / 定位服务 → Animal Poke  
- **Android**：设置 → 应用 → Animal Poke → 权限  
- **浏览器 / PWA**：站点设置中的相机与位置权限  

关闭权限后，已基于该权限处理的信息按《隐私政策》处理；您可进一步申请删除。

## 与敏感权限相关的同意

相机与位置权限的开启，应建立在您已同意《隐私政策》中相应处理规则的基础之上。产品实现上，**先完成应用内知情同意，再请求系统权限**（见 ConsentGate）。
