# Animal Poke SDK 与第三方服务清单

**版本号**：v1.1.0  
**更新日期**：2026-07-10  
**说明**：本清单依据当前仓库实装整理，区分「客户端内嵌 SDK」与「服务端委托调用」。**客户端不集成地图 / 天气 / AI 的第三方 SDK，也不持有第三方 Key**；相关能力均由 Go 后端代理。

## 一、客户端（Web / PWA）

### 1.1 运行时依赖（npm production）

| 名称 | 版本（约） | 类型 | 是否采集/外传个人信息 | 用途 | 官方 / 说明 |
|------|------------|------|------------------------|------|-------------|
| **react** | ^18.3 | UI 框架 | 否（本地渲染） | 界面 | Meta 开源 |
| **react-dom** | ^18.3 | UI 框架 | 否 | DOM 渲染 | 同上 |
| **idb** | ^8 | 本地库 | 否（仅本机 IndexedDB） | 图鉴等本地持久化 | jakearchibald/idb |

**结论：MVP 客户端无广告 SDK、无统计 SDK、无社交登录 SDK、无地图/支付原生 SDK。**

### 1.2 构建期 / PWA 相关（dev 或打包进产物，但不向第三方回传业务数据）

| 名称 | 用途 | 个人信息 |
|------|------|----------|
| Vite 6 + @vitejs/plugin-react | 构建与开发服务器 | 否 |
| **vite-plugin-pwa**（Workbox） | Service Worker、离线静态资源缓存 | 仅缓存本站静态资源；**不缓存鉴权 API**（见工程配置） |
| vite-plugin-compression | gzip/brotli 压缩 | 否 |
| TypeScript / Vitest 等 | 开发与测试 | 否 |

### 1.3 浏览器系统能力（非第三方 SDK，但属敏感权限）

| 能力 | API | 用途 | 说明 |
|------|-----|------|------|
| 相机 | `getUserMedia` | 发现 / 捕获取帧 | 需系统权限 + 隐私同意 |
| 定位 | `Geolocation` | LBS / 逆地理 | 坐标经后端粗化后请求地图服务 |
| 网络 | `fetch` | 仅请求自有后端 `VITE_API_BASE_URL` | 不直连第三方业务 API |
| 本地存储 | `localStorage` / IndexedDB | Token、同意记录、进度 | 设备本地 |

## 二、服务端委托处理的第三方服务（非客户端 SDK）

以下均由 **Go 后端** 使用服务端 Key 调用；配置见 `backend/.env.example`。

| 序号 | 服务 | 配置项 | 接口形态（实装） | 传输的数据 | 目的 | 处理方式 |
|------|------|--------|------------------|------------|------|----------|
| 1 | **腾讯位置服务（腾讯地图）** | `TENCENT_MAP_KEY` | `https://apis.map.qq.com/ws/geocoder/v1/` 逆地理 | 经纬度（服务端可先粗化） | 解析省市区 / 城市 | 委托处理；结果缓存约 6h |
| 2 | **彩云天气** | `CAIYUN_WEATHER_KEY` | `https://api.caiyunapp.com/v2.5/{token}/{lng},{lat}/daily.json` | 经纬度 | 一周天气，驱动玩法与状态 | 委托处理；可缓存 |
| 3 | **视觉大模型 VLM** | `VISION_ENDPOINT` / `VISION_KEY` / `VISION_MODEL`（兼容 `VLM_*`） | OpenAI 兼容 HTTP API（可配置） | **图像二进制（瞬时）** + 提示词 | 动物检测 / 分析 | 推理后服务端 **即时销毁原图**；不入训练（产品策略） |
| 4 | **文本大模型 LLM** | `LLM_ENDPOINT` / `LLM_KEY` / `LLM_MODEL` | OpenAI 兼容 HTTP API（可配置） | 结构化文本 / 分析结果摘要 | 属性与叙事生成 | 委托处理；不回传原始照片（走文本链路） |

**VLM / LLM 说明**：代码为 **多供应商可配置**（Endpoint + Key + Model）。测试用例中出现过 `qwen3.6-flash` 等模型名示例，**不等于已绑定唯一厂商**。上线前请在清单中写明实际签约主体（如通义 / 混元 / 其他 OpenAI 兼容网关）及其隐私政策链接。

### 2.1 基础设施（托管，非玩法 SDK）

| 组件 | 配置 | 数据 | 用途 |
|------|------|------|------|
| **MySQL** | `DB_*` | 设备、进度、审计、同意等 | 主数据库 |
| **Redis**（可选） | `REDIS_URL` | 限流计数等 | 共享限流；未配置则内存实现 |
| 云主机 / 容器 / 对象存储 | 部署侧 | 运行时与备份 | 见部署文档；上线后填写云厂商 |

### 2.2 明确未接入（当前代码无）

| 类型 | 状态 |
|------|------|
| 友盟 / Firebase Analytics / 神策等统计 SDK | ❌ 未接入 |
| 微信 / 支付宝 / 苹果登录 SDK | ❌ 未接入 |
| 推送（APNs / FCM / 厂商推送） | ❌ 未接入 |
| 广告 SDK | ❌ 未接入 |
| 客户端内嵌腾讯地图 JS SDK / 彩云 JS SDK | ❌ 未接入（仅服务端 HTTP） |
| 崩溃监控（Sentry 等） | ❌ 未接入（仅有自研错误上报至自有后端） |

## 三、自有服务（不算第三方 SDK，供对照）

| 名称 | 说明 |

## 四、数据流简图

```text
[用户设备]
  相机/定位权限
  React + idb + PWA(Workbox 仅静态缓存)
        │  仅 HTTPS → 自有后端（Bearer + Request-ID）
        ▼
[Go 后端 · 持有全部第三方 Key]
  ├─→ 腾讯地图（逆地理）
  ├─→ 彩云天气（预报）
  ├─→ VLM（图像瞬时推理 → 销毁）
  └─→ LLM（文本生成属性）
        │
        ▼
     MySQL /（可选）Redis
```

## 五、与用户权利的关系

- 拒绝相机 / 定位：对应第三方 **不会被调用**（或仅浏览模式）。  
- 撤回同意：停止上传图像与非必要定位相关请求。  
- 第三方变更：更新本清单 + 《隐私政策》并依法告知。
