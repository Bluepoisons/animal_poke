# 伙伴远征 · 全中文人工智能剧情系统

## 产品定位

“伙伴远征”取代旧城市手账，直接使用玩家在“动物记录”中拥有的动物。玩家选择伙伴和幻想目的地，操纵动物在轻量地图上移动，到达奇遇点后完成一次由人工智能生成的中文分支剧情。

动物记录由内容注册表驱动：首批提供 35 个常见类别，并以“其他动物”承接模型已经确认、但尚未细分的真实动物。具体名称通过简体中文标签保留，不会把狐狸误归为狗，也不会把普通鸟类默认归为鹅。

核心循环：

```text
选择伙伴 -> 选择幻想区域 -> 人工智能生成中文奇遇 -> 操纵移动 -> 触发事件
         -> 选择行动方式 -> 服务端结算共同回忆 -> 羁绊加深
```

## 交互原则

- 名字可以保留英文；物种、品种、职业、元素、描述、对白、选项和结局必须使用简体中文。
- 三个选择分别表达勇气、好奇与温柔，不存在隐藏最优解，统一增加 6 点羁绊。
- 生成响应不提前下发选项结局、纪念物或羁绊数值，避免剧透与客户端篡改。
- 真实动物照片只在本地作为头像展示，不发送给剧情模型。
- 探险发生在明确的幻想世界，不推断真实动物经历、情绪、健康、主人或精确位置。
- 生产环境人工智能生成失败时返回可重试错误，不用预设故事冒充生成内容，也不增加羁绊。

## 服务端边界

```text
POST /api/v1/adventures
GET  /api/v1/adventures
GET  /api/v1/adventures/:run_id
POST /api/v1/adventures/:run_id/choices
```

服务端从 `Animal` 和 `CompanionProfile` 读取权威上下文，持久化完整 `AdventureRun`。选择完成后，服务端写入固定且幂等的 `companion_memory` 成长事件：

```text
event_id    = adventure:<run_id>:complete
animal_uuid = run.animal_uuid
source_type = adventure
source_id   = run_id
```

客户端只能提交动物 UUID、目的地与选择 ID，不能提交属性、羁绊或奖励值。

## 中文生成约束

- Analyze 阶段要求品种、颜色、体型为简体中文。
- Value 阶段要求动物幻想描述与说明为简体中文。
- Adventure 阶段所有展示字段必须包含中文，并经过字段长度、固定选择 ID、提示词注入和叙事安全校验。
- 英文内部枚举仅用于兼容战斗与数据协议，展示层统一映射为中文。

## 数据与隐私

- 人工智能上下文仅包含动物 UUID 对应的物种、品种、职业、元素、数值与羁绊等级。
- 不发送照片、经纬度、设备 ID、账号 ID 或历史真实位置。
- 探险 run 纳入隐私导出与设备/账号删除流程。
- 旧 `#journal` 与 `#prologue` 仅作兼容跳转，统一进入 `#adventure`。
