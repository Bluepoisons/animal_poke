## 一、规格摘要（对齐当前代码）

**核心循环**：发现 → 捕获 → 收藏（附带战斗、社交、区域竞技）

**MVP / 生产路径**
- 前端：`frontend/src/App.tsx` → `features/animal-poke/AnimalPokeApp`
- 后端：Go（Gin）`backend/`，客户端仅持 `BACKEND_BASE_URL` + 设备 Token
- 体力（代码权威）：`maxStamina` 起始 **120**（Lv.10 **240**），每小时恢复 **10**，单次捕获消耗 **20**  
  （设计文档旧文「10 点体力 + 每日重置」已 superseded by code；勿再写回任务完成标准）
- 本地存储：IndexedDB `animal-poke-db` v3（animals / settings / sync_queue）
- 捕获生成：`services/capturePipeline` + `services/syncQueue`（IDB）

**验收里程碑**：内部可完成「发现→捕获→收藏」全流程（见 Playwright `frontend/e2e/capture-loop.spec.ts` + 单元测试）。
