## 一、规格摘要(直接引自设计文档)

**核心循环**(3.1):发现 → 捕获 → 收藏(附带战斗、社交、区域竞技)

**MVP 交付目标**(7.2):核心循环可玩 Demo,**猫物种优先**
- 云端 VLM 检测接入(猫),端到端延迟 ≤ 300ms,误检 < 8%
- 捕获小游戏原型,投掷手感流畅,单局可完成
- 稀有度生成,5 级可生成,逻辑透明可查
- 体力系统,10 点体力 + 每日重置 + 升级恢复
- 本地存储,动物数据可持久化,重启不丢失
- 基础 UI,发现/捕获/图鉴三屏可用
- 金币 + 道具商店(基础),玩具球可购买使用

**验收里程碑**:内部团队可完整体验"发现→捕获→收藏"全流程。

## 二、技术栈(已确认,引自 2.3 + project.godot)

| 层 | 选型 | 备注 |
|----|------|------|
| 游戏引擎 | **Godot 4.7**(Forward Plus 渲染器) | project.godot 已配置 |
| 物理引擎 | **Jolt Physics 3D** | project.godot 已配置,用于投掷小游戏 |
| UI 框架 | Godot Control 节点 + Theme 主题系统 | 关键界面原生渲染优化 |
| AR 层 | GDExtension 绑定 **ARKit(iOS)/ARCore(Android)** | Godot 无 Unity AR Foundation 等价物,**需自研或社区插件** |
| 云端 AI | 云端 VLM(视觉)+ 云端 LLM(数值/叙事) | 客户端**零本地推理**,仅取帧上传 + 结果接收 |
| 地图 | 腾讯地图 SDK(LBS/定位/逆地理) | key 存 `.env` |
| 天气 | 游戏内按地级市随机生成(3.5);彩云 API 可选 | MVP 不依赖外部天气 |
| 本地存储 | SQLite + 加密 | 动物元数据/进度 |
| 后端 | API 网关 + 鉴权 + 同步 + 反作弊审计 + 弹性扩容 | MVP 单实例,公测上 K8s |
| 架构模式 | **在线优先**(4.3) | 发现/捕获需联网,断网仅浏览图鉴 |
| API Key | `.env` 文件(已 gitignore),环境变量读取 | 禁止硬编码 |

## 五、MVP 阶段任务(核心循环,3 个月,猫优先)

### A. 云端 AI 层

#### [ ] Task M1: 云端 VLM 检测接入(猫)
**Description**:接入云端视觉检测 API,关键帧异步上传,返回检测框 + 物种(猫)+ 置信度。封装统一接口 `VisionClient.detect(frame) -> {boxes, species, confidence}`,便于后续切换 provider(通义千问-VL / GLM-4V / GPT-4o vision)。
**Acceptance Criteria**:
- 上传一帧返回结构化 JSON
- 端到端延迟 ≤ 300ms
- 置信度 ≥ 0.85 触发发现事件
- 结果缓存(同会话复用)

**Files**: `scripts/core/ai/vision_client.gd`
**Reference**: 3.2 检测流程、4.2 模型选型、4.5 照片瞬时推理不落盘

#### [ ] Task M2: 云端 VLM 深度分析 + LLM 数值生成
**Description**:捕获成功后单次调用 VLM(品种/毛色/体型/质量/角度评分)+ LLM(稀有度编排 + 六维属性 + 生成依据叙事文本)。稀有度 = f(品种权重,毛色权重,拍摄质量,物种权重,随机种子),映射 5 级。拍摄质量 6 维评分(主体完整度/清晰度/光线/构图/姿态/拍摄角度)。
**Acceptance Criteria**:
- 5 级稀有度可生成,映射符合 5.3 公式
- 六维属性(HP/ATK/DEF/SPD/Class/Element)范围符合 5.2
- 生成依据(四项得分明细 + 叙事)可查
- 随机种子可追溯

**Files**: `scripts/core/ai/analysis_client.gd`、`scripts/core/ai/value_generator.gd`
**Reference**: 3.6 数值生成、5.1-5.3、5.2 六维属性

### B. 发现模块

#### [ ] Task M3: 相机取景 + 关键帧采样 + 上传
**Description**:相机预览帧自适应 2~5 FPS 抽帧上传;命中后用设备运动传感器做 AR 锚定跟踪,减少持续上传。记录 GPS/时间戳/设备姿态/置信度/推理请求 ID。
**Acceptance Criteria**:
- 帧采样频率可配,控带宽
- AR 锚定命中后停止持续上传
- 审计字段完整记录

**Files**: `scripts/modules/discover/camera_controller.gd`、`frame_sampler.gd`、`ar_anchor.gd`
**Reference**: 3.2 检测流程 1-4 步

#### [ ] Task M4: 防作弊多重校验
**Description**:帧时间戳连续性校验(防抽帧注入)、EXIF 与设备传感器交叉验证、检测框运动轨迹合理性(防静态图片翻拍)、root/越狱检测(降级仅浏览模式)。禁止相册导入/屏幕录制/模拟摄像头。
**Acceptance Criteria**:
- root/越狱设备降级为仅浏览
- 翻拍静态图片被标记可疑
- 异常抛物线/轨迹记录审计

**Files**: `scripts/modules/discover/anti_cheat.gd`、`scripts/core/security/device_check.gd`
**Reference**: 3.2 防作弊多重校验、4.5、九-防作弊风险

#### [ ] Task M5: 弱网降级(扫描式)
**Description**:弱网时降级为玩家手动点按扫描键,单帧上传云端判定。断网时明确提示"网络异常,发现/捕获暂停",仅保留图鉴浏览。
**Acceptance Criteria**:
- 弱网自动切扫描模式
- 断网有明确提示,不误操作
- 网络恢复自动续接

**Files**: `scripts/modules/discover/network_mode.gd`
**Reference**: 3.2 检测流程 5、4.3 在线优先

### C. 捕获小游戏

#### [ ] Task M6: 3D 物理投掷核心(猫)
**Description**:Jolt Physics 3D 投掷。滑动手势决定方向与力度,借鉴愤怒小鸟抛物线手感。投掷物=猫粮罐。目标=AR 锚定位置。基础成功率 70%,受警惕值(距离/动作幅度)影响。每次消耗 1 食物 + 1 体力。
**Acceptance Criteria**:
- 投掷手感流畅 60fps
- 单局 ≤ 90s
- 命中 → 食物散开动画 → 进入收服判定
- 体力/食物正确消耗

**Files**: `scripts/modules/capture/throw_controller.gd`、`projectile.gd`、`capture_scene.tscn`
**Reference**: 3.3 捕获小游戏

> **PM 注**:10.1 的连击系统/命中梯度/动物行为变化属于"玩法深化",MVP 先做基础二元命中(中/不中),避免 scope creep。深化放内测阶段。

#### [ ] Task M7: 投掷反作弊
**Description**:投掷轨迹需符合物理参数范围,异常抛物线(瞬时位移)标记可疑。
**Acceptance Criteria**:
- 异常轨迹标记并记录推理请求 ID

**Files**: `scripts/modules/capture/trajectory_validator.gd`
**Reference**: 3.3 反作弊

### D. 收藏模块

- **战斗系统 v1**:自动回合制 PvE(6.5),3v3 按 SPD 顺序,元素克制
- **捕获深化**(10.1):连击、命中梯度、动物行为变化
- **亲密度 + 区域声望**(5.6):双轨制
- **等级完整 10 级 + 成就系统**(5.5、6.6)
- **每日商店随机刷新**(6.4,依赖等级加成)

## 九、质量要求(硬性)

- [ ] **无后台进程**:任何命令禁止追加 `&`,禁止启动常驻服务
- [ ] **性能基线**:中端机(骁龙6系/A12)≥ 30fps,高端机 ≥ 60fps;单次发现→捕获 ≤ 5 分钟
- [ ] **包体**:安装包 ≤ 150MB(无端侧模型,仅引擎+资源)
- [ ] **崩溃率**:< 0.5%
- [ ] **移动适配**:iOS 14+/Android 8+,刘海屏/灵动岛安全区,@2x/@3x 资源
- [ ] **在线优先**:发现/捕获/数值生成必须联网;断网仅图鉴浏览,有明确提示
- [ ] **隐私合规**:照片上传云端瞬时推理后即时销毁,不落盘不入训练,首次授权弹窗,可撤回
- [ ] **API Key**:统一 `.env`,禁止硬编码,`.env` 在 `.gitignore`
- [ ] **数据可溯源**:每只动物保留推理请求 ID + 模型版本 + 时间戳链路
- [ ] **图鉴虚拟化渲染**:防 CatchCat 滚动卡顿

## 十、技术说明与 PM 风险提示

**关键技术风险(MVP 阶段优先验证)**:
1. **AR 层自研成本**:Godot 无 Unity AR Foundation 等价物,GDExtension 绑定 ARKit/ARCore 需自研或找社区插件——这是 MVP 最大不确定项,建议 F 阶段就做技术预研(POC),不要等到写发现模块才发现卡住。
2. **云端 VLM 延迟**:端到端 ≤ 300ms 是硬指标,受网络 + 模型推理双重影响。关键帧采样 + 结果缓存是核心优化手段。建议 M1 最早接入做压测。
3. **云端推理成本**:随 DAU 线性增长。MVP 就要建立单用户每日调用上限 + 成本告警,否则上线即亏钱。
4. **防作弊 vs 体验平衡**:多重校验不能误伤正常玩家(弱网、低端机传感器精度差)。降级路径要平滑。

**Scope 控制原则**:
- 第十章"玩法深化"全部**不在 MVP**,内测起逐步引入
- 双货币硬货币、商业化**不在 MVP**
- 多语言框架**前置**(文案与代码分离),但 MVP 只做中文
- 天气系统 MVP 可不做(内测引入),但发现屏先预留天气提示位

**排期现实预期**:
- Foundation 2-3 周(含 AR POC)
- MVP 3 个月(猫物种单物种跑通)
- 首版实现通常需 2-3 轮修订,不要追求一次到位
