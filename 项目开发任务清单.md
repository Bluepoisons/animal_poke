## 二、技术栈(已确认,引自 2.3 + project.godot)

| 层 | 选型 | 备注 |
|----|------|------|
| 游戏引擎 | **Godot 4.7**(Forward Plus 渲染器) | project.godot 已配置 |
| 物理引擎 | **Jolt Physics 3D** | project.godot 已配置,用于投掷小游戏 |
| UI 框架 | Godot Control 节点 + Theme 主题系统 | 关键界面原生渲染优化 |
| AR 层 | GDExtension 绑定 **ARKit(iOS)/ARCore(Android)** | Godot 无 Unity AR Foundation 等价物,**需自研或社区插件** |
| 云端 AI | 云端 VLM(视觉)+ 云端 LLM(数值/叙事) | 客户端**零本地推理,零直连**;**经 Go 后端编排调用** |
| **后端服务** | **Go**(Gin/Echo)+ API 网关 | **联网服务总枢纽**:鉴权/同步/反作弊/区域排行/对象存储 + 全部第三方 API 代理与 AI 推理编排;MVP 单实例,公测上 K8s |
| 地图 | 腾讯地图(逆地理/POI) | **经 Go 后端代理调用**,Key 只存后端 `.env`;客户端定位走系统能力 |
| 天气 | 游戏内按地级市随机生成(3.5);彩云 API 可选 | MVP 不依赖外部天气;彩云**经后端代理** |
| 本地存储 | SQLite + 加密 | 动物元数据/进度 |
| 架构模式 | **在线优先**(4.3) | 客户端只与 Go 后端通信;发现/捕获需联网,断网仅浏览图鉴 |
| API Key | 第三方 Key 只存**后端** `.env`;客户端仅持后端地址 + 设备 Token | 禁止硬编码,`.env` 在 `.gitignore` |

### C. 捕获小游戏

#### [ ] Task M6: 3D 物理投掷核心(猫)
**Description**:Jolt Physics 3D 投掷。滑动手势决定方向与力度,借鉴愤怒小鸟抛物线手感。投掷物=猫粮罐。目标=AR 锚定位置。基础成功率 70%,受警惕值(距离/动作幅度)影响。每次消耗 1 食物 + 1 体力。

## 十、技术说明与 PM 风险提示

**关键技术风险(MVP 阶段优先验证)**:
1. **AR 层自研成本**:Godot 无 Unity AR Foundation 等价物,GDExtension 绑定 ARKit/ARCore 需自研或找社区插件——这是 MVP 最大不确定项,建议 F 阶段就做技术预研(POC),不要等到写发现模块才发现卡住。
2. **云端 VLM 延迟**:端到端 ≤ 300ms 是硬指标,受网络 + 模型推理双重影响。关键帧采样 + 结果缓存是核心优化手段。建议 M1 最早接入做压测。
3. **云端推理成本**:随 DAU 线性增长。MVP 就要建立单用户每日调用上限 + 成本告警,否则上线即亏钱。
4. **防作弊 vs 体验平衡**:多重校验不能误伤正常玩家(弱网、低端机传感器精度差)。降级路径要平滑。

**Scope 控制原则**:
- 第十章"玩法深化"全部**不在 MVP**,内测起逐步引入
- 双货币硬货币、商业化**不在 MVP**
- 多语言框架**前置**(文案与代码分离),但 MVP 只做中文
- 天气系统 MVP 可不做(内测引入),但发现屏先预留天气提示位

**排期现实预期**:
- Foundation 2-3 周(含 AR POC)
- MVP 3 个月(猫物种单物种跑通)
- 首版实现通常需 2-3 轮修订,不要追求一次到位
