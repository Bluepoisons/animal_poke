# 概览:LBS 动物收集手游 开发任务清单

## 做了什么
基于 `游戏开发计划.md` v1.2(2026-07-08)与已确认技术栈(Godot 4.7 + Jolt Physics + 云端 VLM/LLM + 在线优先),将完整设计文档转化为可执行的开发任务清单。

## 产出文件
- `项目开发任务清单.md` — 执行层任务清单(与设计文档分离)

## 关键决策
1. **职责分离**:设计文档(`游戏开发计划.md`)是设计单一事实来源,不碰;新文件是执行层任务清单,职责清晰。符合项目"单一文件"约定(该约定针对设计散落,不禁止任务清单)。
2. **新增 Foundation 阶段**:project.godot 当前基本是空壳(仅基础配置+Jolt),没有任何代码骨架。在写业务模块前先搭架构地基(目录/Autoload/Theme/DB/日志),2-3 周,避免反复重构。
3. **聚焦 MVP 细粒度**:14 个 MVP 任务(M1-M14)拆到 30-60 分钟可动手粒度;内测/公测/上线做高层级规划(依赖 MVP 学习成果)。符合 PM"不镀金、先功能后打磨"原则。
4. **Scope 控制**:第十章"玩法深化"(连击/半自动战斗/养成/PVE/社交/赛季)全部不在 MVP;双货币硬货币、商业化不在 MVP;天气系统 MVP 可不做但预留 UI 位。
5. **风险前置**:标注 4 个 MVP 关键技术风险(AR 层自研成本、VLM 延迟、推理成本、防作弊 vs 体验),建议 AR POC 在 Foundation 阶段就做。

## 后续

- 云端 VLM provider 选型(通义千问-VL / GLM-4V / GPT-4o)需采购评估,国内首发优先国产
- 任务系统已录入 10 个主要任务组,开发推进时更新状态
