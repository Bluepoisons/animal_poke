# LBS 动物收集手游 · 开发计划

> 基于 CatchCat 设计理念改进，规避其后端无扩容预案、品种判定不透明、UI 卡顿等已暴露问题。

---

## 一、游戏定位

| 维度 | 说明 |
|------|------|
| 类型 | LBS（基于位置服务）收集类手游 |
| 核心体验 | 真实世界探索 + 云端 VLM 实时动物识别 + 云端 LLM 生成数值 |
| 核心循环 | **发现 → 捕获 → 收藏**（附带战斗、社交、区域竞技） |
| 动物种类 | 首期支持 **猫 / 鹅 / 狗** 三类，架构预留可扩展更多物种 |
| 差异化 | 1) 真实动物 AR 识别，非虚拟随机生成；2) 数值透明可溯源；3) 云端 AI 全链路推理 + 隐私严格保护；4) 区域竞技 + 天气动态玩法 |
| 目标用户 | 动物爱好者、户外探索玩家、轻度收集向玩家、口袋妖怪类怀旧人群 |
| 目标市场 | 国内首发，预留多语言（中日英）扩展能力 |

### 与 CatchCat 的关键差异

| CatchCat 暴露问题 | 本方案应对策略 |
|---|---|
| 后端无扩容预案，峰值崩溃 | Go 后端统一承接联网请求 + 弹性扩容，并编排云端 AI 推理、处理元数据同步与第三方 API 代理 |
| 品种判定不透明，用户质疑公平性 | 公开稀有度算法白皮书，每只动物附带生成依据可查 |
| UI 卡顿严重 | 原生渲染关键路径 + 推理卸载至云端，本地不占算力，保证 60fps |

## 二、目标平台

### 2.1 平台策略

- **主平台**：移动端（iOS + Android）
- **最低系统**：iOS 14.0+ / Android 8.0 (API 26)+
- **推荐系统**：iOS 16.0+ / Android 10.0+（用于完整 ARCore/ARKit 体验）

### 2.2 适配要求

| 项目 | 要求 |
|------|------|
| 屏幕尺寸 | 4.7" ~ 6.9" 全覆盖，iPad 兼容运行（非优先）|
| 分辨率 | @2x / @3x 资源适配，刘海屏 / 灵动岛安全区处理 |
| 性能基线 | 中端机（骁龙 6 系 / A12）≥ 30fps，高端机 ≥ 60fps |
| 网络容忍 | 需稳定网络（云端 VLM/LLM 推理依赖在线）；断网时仅可浏览本地图鉴 |
| 续航控制 | 单次完整发现→捕获流程 ≤ 5 分钟，相机持续运行耗电 ≤ 8%（无本地推理发热）|
| 存储占用 | 安装包 ≤ 150MB（无端侧模型，仅引擎+资源），后续按需下载美术资源包 |

### 2.3 技术栈选型

- **游戏引擎**：Godot 4.7
- **UI 框架**：Godot Control 节点体系 + Theme 主题系统（关键界面走原生渲染优化）
- **AR 层**：iOS 走 ARKit、Android 走 ARCore，通过 GDExtension 绑定平台原生 AR 能力（Godot 无 Unity AR Foundation 等价物，需自研或社区插件接入）
- **后端服务（Go）**：**所有联网服务统一收口到 Go 后端**，包括鉴权、同步、反作弊审计、区域排行聚合、对象存储，以及对全部第三方 API（腾讯地图 / 彩云天气 / 云端 VLM / 云端 LLM）的**统一代理与编排**。客户端**不持有任何第三方 API Key，不直接调用任何外部服务**，只与 Go 后端通信
- **云端 AI 推理**：云端 VLM（视觉理解：检测/品种/质量评分）+ 云端 LLM（数值生成/派遣叙事），**由 Go 后端编排调用**，客户端仅负责取帧上传至后端、接收后端回传的结构化结果，**不做任何本地推理，也不直连 VLM/LLM**
- **天气数据**：游戏内按地级市随机生成（见 3.5）；彩云天气 API 作为可选真实天气源，**经 Go 后端代理调用**（key 只存后端 .env，客户端不接触，见 4.6）

### 3.2 发现机制

**输入约束（防作弊核心）**：
- ✅ 仅接受 **实时相机取景帧** 作为输入
- ❌ 禁止相册导入、屏幕录制、模拟摄像头
- ❌ 禁止 root/越狱设备（检测到则降级为"仅浏览"模式）

**检测流程**（云端 VLM，自适应帧采样平衡带宽/成本/延迟）：
1. 相机预览帧以 **2~5 FPS** 抽取关键帧上传云端视觉模型（非逐帧上传，控带宽与调用成本）
2. 云端模型输出动物检测框 + 物种分类（猫/鹅/狗）+ 置信度；命中后客户端用设备运动传感器做帧间 AR 锚定跟踪，减少持续上传
3. 置信度 ≥ 0.85 时触发"发现"事件，进入捕获状态
4. 同步记录：GPS 坐标、时间戳、设备姿态、检测置信度、云端推理请求 ID（用于后续反作弊审计）
5. 弱网时降级为"扫描式"交互：玩家手动点按扫描键，单帧上传云端判定

**防作弊多重校验**：
- 帧时间戳连续性校验（防止抽帧注入）
- EXIF 元数据与设备传感器数据交叉验证
- 检测框运动轨迹合理性（防止静态图片对屏幕翻拍）

### 3.3 捕获小游戏

**玩法**：3D 物理投掷对应食物

| 动物 | 投掷物 | 说明 |
|------|--------|------|
| 猫 | 猫粮罐 | 标准抛物线手感 |
| 鹅 | 面包屑球 | 弹跳略强 |
| 狗 | 骨头零食 | 下落更快 |

| 元素 | 设计 |
|------|------|
| 操作 | 滑动手势决定投掷方向与力度，借鉴"愤怒的小鸟"抛物线手感 |
| 目标 | 命中场景中识别到的动物（AR 锚定位置）|
| 时机 | 动物会移动/警戒，需预判投掷点 |
| 基础成功率 | 70%，受动物"警惕值"（距离、动作幅度）与**天气**影响（见 3.5）|
| 道具增益 | 使用玩具球可 +15% 成功率（见经济系统）|
| 资源消耗 | 每次投掷消耗 1 个对应食物 + 1 点体力 |
| 反馈 | 命中 → 食物散开动画 → 进入"收服"判定 → 数值生成 |

**反作弊**：投掷轨迹需符合物理参数范围，异常抛物线（如瞬时位移）标记为可疑。

### 3.4 体力系统

**体力是玩家行动的核心资源**，用于捕获动物与派遣宠物淘金币。

| 项目 | 规则 |
|------|------|
| 体力上限 | 初始 120 点，随等级成长至满级 240 点（见 5.5）|
| 单次消耗 | 捕获 1 次 = 20 体力；派遣淘金币 = 20 体力/次 |
| 自然恢复 | **每小时 +10 点**（即每 6 分钟 +1 点），不超过上限 |
| 升级恢复 | 升级时恢复满体力 |
| 体力为 0 | 无法捕获/派遣，但仍可浏览图鉴、社交、管理宠物 |
| 体力购买 | 可用金币购买额外体力（每日限购 3 次，防沉迷）|

**数值平衡说明**：
- 满体力 120 点 = 可连续捕获 6 次；满级 240 点 = 12 次
- 每小时恢复 10 点 = 每 6 分钟 +1 点，满体力（120）需 12 小时回满
- 鼓励玩家分时段上线，而非一次性耗尽
- 体力双用途（捕获 / 淘金币）形成策略取舍

### 3.5 天气系统

**定位**：天气**仅影响捕捉难易度与宠物状态**，**不影响动物刷新与稀有度**。动物出现率由 LBS 探索与诱饵道具决定，保持公平可预期。

**生成机制**（不依赖外部天气 API）：
- 按**地级市**划分，每个地级市独立拥有本周天气
- **每周日 0:00 结算并刷新**下一周天气，持续整周（周一至周日）
- 天气由游戏内**随机**生成，极端天气（雪/雾/极端）概率较低

**随机概率表**（极端天气概率较低，示例可调）：

| 天气 | 概率 |
|------|------|
| 晴 | 33% |
| 多云 | 24% |
| 阴 | 18% |
| 雨 | 15% |
| 雪 | 4% |
| 雾 | 4% |
| 极端天气 | 2% |

**天气影响表**：

| 天气 | 捕捉难易度 | 宠物状态影响 |
|------|-----------|-------------|
| 晴 | 光线充足、检测精准，基础成功率 **+5%** | ☀️ 心情愉悦，全属性 **+5%**（见 5.4 愉悦状态）|
| 多云 | 标准 | 标准 |
| 阴 | 光线偏暗，基础成功率 **-5%** | 标准 |
| 雨 | 视线受阻、地面湿滑，基础成功率 **-15%** | ⚠️ 出战宠物 **8% 概率感冒**（见 5.4）|
| 雪 | 低温影响操作，基础成功率 **-10%** | ⚠️ 出战宠物 **6% 概率感冒**（见 5.4）|
| 雾 | 检测距离缩短，基础成功率 **-10%** | 标准 |
| 极端天气 | 暂停户外玩法 | 提示玩家注意安全 |

**感冒触发时机**：本周天气为雨/雪时，出战宠物按调低后的概率感冒（缓解一周固定恶劣天气的累积风险）。
**愉悦触发时机**：晴天出战/参战的宠物获得愉悦 buff，**本周有效**（因天气一周固定）。
**地域探索动机**：玩家跨城市移动会切换到目标城市的本周天气——可去晴天城市享愉悦，或主动去雨天城市挑战感冒相关玩法。
**UI 提示**：主界面显示"当前城市 · 本周天气"（如"宁波 · 晴朗"）。

### 3.6 数值生成

- 捕获成功后，**云端 VLM** 分析动物的品种、毛色、体型、拍摄质量、拍摄角度
- **云端 LLM** 结合分析结果与随机种子，生成该动物的稀有度、六维属性，并输出可溯源的"生成依据"叙事文本
- **全程云端完成**，客户端仅接收生成结果的元数据（稀有度/属性/品种/依据文本），原始照片按隐私策略处理（见 4.5）

### 4.2 云端 AI 模型选型

> 全部为云端模型，客户端不内置任何推理模型。选型为示意，具体 provider 以采购评估为准（国内首发优先国产可用方案）。

| 模型类型 | 用途 | 选型示意 | 调用方式 |
|---------|------|---------|---------|
| 云端视觉检测 | 发现阶段多物种检测（猫/鹅/狗）+ 检测框 | 轻量视觉检测 API（如通义千问-VL 检测能力 / GPT-4o vision / 专用目标检测接口）| 客户端关键帧上传**至 Go 后端**，后端异步调 VLM，返回框 + 物种 + 置信度 |
| 云端 VLM（深度分析）| 捕获后品种识别、毛色分类、体型、拍摄质量、拍摄角度评分 | 通义千问-VL / GLM-4V / GPT-4o vision | 捕获成功后客户端单次请求后端，**Go 后端调 VLM**，结构化 JSON 输出 |
| 云端 LLM（数值/叙事）| 稀有度算法编排、六维属性生成、生成依据叙事文本、派遣任务叙事 | 通义千问 / GLM-4 / DeepSeek / GPT-4o-mini | **Go 后端**接收 VLM 分析结果 + 随机种子，调 LLM 返回数值 + 可溯源依据 |

**优化策略**：
- **关键帧采样**：发现阶段非逐帧上传，自适应 2~5 FPS 抽帧，命中后靠设备运动做 AR 锚定跟踪，显著降低带宽与调用成本
- **结果缓存**：同一捕获会话内复用检测结果，避免重复 VLM 调用
- **结构化输出**：VLM/LLM 统一返回 JSON，便于解析与防作弊校验
- **分级调用**：发现走轻量检测 API（快/省）；深度分析仅在捕获成功后调用一次 VLM + 一次 LLM，控制成本
- **服务端 Prompt 工程**：Prompt 模板集中管理，支持 A/B 与迭代，不依赖客户端版本
- **成本监控**：按 DAU 预估调用量，设置单用户每日调用上限与告警

### 4.3 在线优先架构

```
在线状态：取景帧上传 Go 后端 ── 后端编排 VLM/LLM 推理 ── 结果回传客户端 ── 元数据落库同步 ──▶ 完成
									│
弱网/断网：核心发现/捕获不可用 ── 仅本地图鉴浏览/宠物管理可离线
									│
					恢复网络：自动恢复在线流程，无增量补传需求
```

**原则**（因 AI 推理全在云端且经 Go 后端编排，架构由"离线优先"调整为"在线优先"）：
- **发现、捕获、数值生成**必须在线（依赖 Go 后端编排云端 VLM/LLM）
- **离线可用**：本地图鉴浏览、宠物管理、已缓存数据查看
- **社交、排行榜、世界事件、天气查询**需在线（均经 Go 后端）
- 离线时给出明确提示"网络异常，发现/捕获暂停"，避免玩家误操作
- 弱网下发现降级为"扫描式"单帧判定（见 3.2 检测流程第 5 步），单帧仍上传后端
- 天气数据由 Go 后端按地级市生成（3.5），客户端在线时经后端拉取本周天气缓存

### 4.4 同步方案

- 每只动物生成唯一 UUID + 设备 ID + 时间戳
- 同步内容：UUID、稀有度、属性、品种、物种、捕获坐标（可选脱敏）、生成时间、云端推理请求 ID
- **原始照片**：按 4.5 隐私策略，上传**至 Go 后端**，由后端转发 VLM 推理后**即时销毁，不持久化存储**（玩家主动分享除外）；客户端不直连 VLM
- 冲突策略：以生成时间戳为准，Go 后端只做去重与校验，不修改数据

### 4.5 隐私保护策略

> **策略调整说明**：因采用云端 VLM，原始照片必须上传推理。照片链路为「客户端 → Go 后端 → 云端 VLM → 后端即时销毁」，客户端不直连 VLM。本方案采用「接受上传 + 严格保护」路线：照片仅瞬时推理、不落盘存储、不入训练、可审计、首次授权明确告知。

| 数据类型 | 处理方式 |
|---------|---------|
| 原始照片 | **上传 Go 后端，由后端转发云端 VLM 瞬时推理，推理完成后后端即时销毁，不持久化存储、不入训练集**；仅保留推理产出的结构化结果（品种/质量分等） |
| 人脸/车牌等敏感信息 | 上传前客户端做人脸模糊预处理；VLM 仅识别动物相关特征 |
| GPS 坐标 | 可选上传，默认仅保留网格化坐标（精度降至 100m）|
| 动物元数据 | 加密传输，仅含数值与品种，不含生物特征 |
| 推理请求 ID | Go 后端留存用于反作弊审计与可溯源，含调用时间/模型版本/置信度，**不含原图** |
| 玩家行为日志 | 匿名化后用于运营分析，可一键导出/删除 |
| 玩家主动分享 | 玩家主动上传分享的缩略图经独立授权后由后端存入对象存储 |
| 合规 | 遵循《个人信息保护法》，首次启动明确授权弹窗，**单独告知照片上传与即时销毁机制**，提供撤回授权入口 |

**严格保护四要点**：
1. **不落盘**：Go 后端内存态转发推理，请求结束即释放，原图不写对象存储
2. **不入训练**：与模型供应商签署协议，禁止用上传图片训练/微调模型
3. **可审计**：Go 后端保留推理请求 ID + 模型版本 + 时间戳链路，供合规审计追溯
4. **知情同意**：首次启动弹窗明确告知"为识别动物品种与拍摄质量，照片将经后端上传云端瞬时处理并立即销毁"，玩家可拒绝（则降级为"仅浏览"模式）

### 4.6 API 接入说明

> **核心原则**：**所有第三方 API Key 只存 Go 后端的 `.env`，客户端不持有、不接触任何第三方 Key**。客户端仅持后端下发的设备 Token 与后端服务地址，所有外部调用（腾讯地图 / 彩云天气 / 云端 VLM / 云端 LLM）一律由 Go 后端代理转发。这样既避免客户端被反编译泄露密钥，也便于在后端集中做限流、审计、provider 切换与成本控制。

#### 4.6.1 Key 管理规范

- **后端 `.env`**：腾讯地图 / 彩云天气 / 云端 VLM / 云端 LLM 等全部第三方 Key 统一存 Go 后端 `.env`（**已加入 .gitignore，切勿提交 git**）
- 后端代码通过环境变量读取，禁止硬编码；团队提供 `.env.example`（不含真实 key）供参考
- **客户端配置**：客户端仅含 `BACKEND_BASE_URL` 与登录后获得的设备 Token，**不含任何第三方 Key**；客户端 `.env` 只存后端地址等非敏感配置

#### 4.6.2 Go 后端对外端点（客户端调用）

客户端所有联网请求都打到 Go 后端，典型端点：

| 端点 | 方法 | 用途 | 对应后端编排 |
|------|------|------|------------|
| `/auth/device` | POST | 设备注册/换取 Token | 后端签发设备 Token |
| `/geo/city` | GET | GPS → 地级市 | 后端调腾讯地图逆地理编码 |
| `/weather/week` | GET | 拉取本周天气 | 后端按地级市生成（3.5），可选叠加彩云 |
| `/vision/detect` | POST | 上传关键帧 → 检测框+物种+置信度 | 后端调云端 VLM（发现阶段）|
| `/vision/analyze` | POST | 上传捕获帧 → 品种/质量/角度评分 | 后端调云端 VLM（深度分析）|
| `/value/generate` | POST | 生成稀有度+六维+叙事 | 后端调云端 LLM |
| `/sync/animal` | POST | 上传动物元数据 | 后端去重校验落库 |
| `/rank/region` | GET | 区域排行 | 后端聚合结算 |

#### 4.6.3 腾讯地图 API（后端代理调用）

| 能力 | 用途 | 后端调用端点 |
|------|------|------|
| 逆地理编码 | GPS → 地级市（天气按地级市划分要用）| `https://apis.map.qq.com/ws/geocoder/v1/?location={lat},{lng}&key={key}` |
| POI 搜索 | 找公园/绿地等动物密集区（可选）| `https://apis.map.qq.com/ws/place/v1/search` |

> 客户端定位（获取 GPS）仍走系统定位能力，不涉及 Key；地图 SDK 展示如需 Key 则由后端签发短期票据，不把长期 Key 下发客户端。逆地理编码示例（后端执行）：`GET .../geocoder/v1/?location=29.87,121.54&key=$TENCENT_MAP_KEY` → 返回 `ad_info.city`（如"宁波市"）。

#### 4.6.4 彩云天气 API（可选，后端代理调用）

当前计划天气走游戏内随机（3.5），彩云作为可选真实源备用，由 Go 后端代理调用：

- 文档：https://docs.caiyunapp.com/weather-api/
- 端点格式：`https://api.caiyunapp.com/v2.6/{token}/{经度},{纬度}/{endpoint}.json`
- 实时：`/realtime.json`，预报：`/hourly.json`、`/daily.json`，全部：`/weather.json`
- 关键字段：`result.realtime.skycon`（天气状况）、`result.realtime.temperature`（温度）

| skycon 值 | 含义 | 映射游戏天气 |
|-----------|------|------------|
| CLEAR | 晴 | 晴 |
| PARTLY_CLOUDY | 多云 | 多云 |
| CLOUDY | 阴 | 阴 |
| RAIN / LIGHT_RAIN 等 | 雨 | 雨 |
| SNOW / LIGHT_SNOW 等 | 雪 | 雪 |
| FOG | 雾 | 雾 |

调用示例（后端执行）：`GET https://api.caiyunapp.com/v2.6/$CAIYUN_WEATHER_KEY/121.54,29.87/realtime.json` → `skycon="CLEAR"` → 游戏天气=晴

#### 4.6.5 调用链路

**客户端视角**（零第三方 Key）：客户端取 GPS → `GET /geo/city` 拿地级市 → `GET /weather/week` 拿本周天气 → 发现/捕获时帧上传 `/vision/*` 与 `/value/generate`。

**后端视角**（持全部 Key）：腾讯逆地理编码拿地级市 →（若用真实天气）彩云查 skycon 映射游戏天气；帧上传后后端调 VLM/LLM 编排推理，结果回传客户端。当前天气走游戏内随机（3.5），彩云作为可选真实源备用。

> 具体参数与返回字段以腾讯地图 / 彩云天气官方文档为准。

## 五、数值系统

### 5.1 稀有度体系

| 等级 | 名称 | 掉率（基础）| 视觉标识 |
|------|------|-----------|---------|
| 1 | Common 普通 | 60% | 灰色边框 |
| 2 | Uncommon 非凡 | 25% | 绿色边框 |
| 3 | Rare 稀有 | 10% | 蓝色边框 |
| 4 | Epic 史诗 | 4% | 紫色边框 |
| 5 | Legendary 传说 | 1% | 金色边框 + 粒子特效 |

### 5.2 六维战斗属性

| 属性 | 缩写 | 说明 | 范围 |
|------|------|------|------|
| 生命 | HP | 承伤上限 | 50~500 |
| 攻击 | ATK | 每回合伤害 | 10~120 |
| 防御 | DEF | 减伤系数 | 5~80 |
| 速度 | SPD | 行动顺序判定 | 1~100 |
| 定位 | Class | 坦克/输出/辅助/控制（4 类）| 枚举 |
| 元素 | Element | 火/水/草/光/暗（5 类，含克制关系）| 枚举 |

**元素克制**：火>草>水>火（循环），光⇄暗（互克，1.5 倍）

**宠物属性差异化**：

不同物种/品种有基础属性偏向，结合六维属性形成"品种特性 + 个体差异"两层结构：

| 物种 | 基础偏向 | 典型表现 |
|------|---------|---------|
| 猫 | 高 SPD、高暴击 | 速度快、出手频繁，脆皮 |
| 狗 | 高 HP、高 ATK | 血厚攻高，坦/输出兼备 |
| 鹅 | 高 DEF、控制型 | 高防御、技能带控制效果 |

**品种特性**：同一物种内不同品种（如猫的英短/布偶/橘猫）在基础六维上有 ±10% 的浮动，形成"同一物种、不同流派"的多样性。例如：
- 英短：HP +10%，DEF +10%，偏坦克
- 布偶：ATK +5%，SPD +10%，偏敏捷输出
- 橘猫：HP +15%，ATK -5%，偏血牛

**个体属性随机**：在品种特性基础上，每只宠物的六维属性有 ±15% 的随机浮动，确保"没有两只完全相同的宠物"。

### 5.3 稀有度分配逻辑（透明化）

```
稀有度 = f(品种稀有权重, 毛色稀有权重, 拍摄质量分, 物种权重, 随机种子)

例：
  基础分 = 品种权重 × 0.35 + 毛色权重 × 0.25 + 拍摄质量 × 0.25 + 物种权重 × 0.15
  最终稀有度 = 基础分 + 随机扰动(±0.1)

映射：
  < 0.3  → Common
  0.3~0.6 → Uncommon
  0.6~0.8 → Rare
  0.8~0.95 → Epic
  ≥ 0.95 → Legendary
```

**拍摄质量评分维度**（6 维 × 0~2 分 = 0~12 分，参考聊天记录"不同角度拍摄有不同分数"）：

| 维度 | 2 分 | 1 分 | 0 分 |
|------|------|------|------|
| 主体完整度 | 全身入镜 | 半身 | 仅头部/局部 |
| 清晰度 | 对焦清晰 | 一般 | 模糊 |
| 光线 | 充足均匀 | 一般 | 过暗/过曝 |
| 构图 | 主体居中占比大 | 一般 | 偏远/过小 |
| 姿态可辨 | 正面/侧面清晰 | 一般 | 难辨 |
| 拍摄角度 | 正面/正侧面（最佳识别角度）| 背面/斜侧面 | 遮挡/无法识别 |

**拍摄角度说明**：根据聊天记录中"不同角度会影响分数"的讨论，云端 VLM 对正面/正侧面的识别准确率最高（正面≈95%、正侧面≈90%、背面≈70%），因此拍摄角度直接影响"品种识别置信度"，进而影响稀有度权重。鼓励玩家从最佳角度拍摄。

**拍摄质量 → 属性影响**（拍得差属性差一点）：

| 质量分 | 稀有度影响 | 六维属性影响 |
|--------|----------|------------|
| 8~10 优 | 稀有度权重 +10% | 全属性 +10% bonus |
| 5~7 良 | 标准 | 标准 |
| 3~4 中 | 稀有度权重 -10% | 全属性 -10% |
| 0~2 差 | 稀有度上限降一级 | 全属性 -25% |

> 拍摄质量同时影响稀有度（通过上方公式权重）与六维属性（直接增减），鼓励玩家认真拍摄而非随手糊弄——这也正面回应了 CatchCat"评分标准不透明"的诟病。

**透明性设计**：
- 每只动物的图鉴页可查看"生成依据"（品种/毛色/质量/物种四项得分明细）
- 算法白皮书在官网公开，用户可复现验证
- 随机种子可追溯（防后台暗箱调率质疑）

### 5.4 状态系统（新增）

宠物可处于以下状态，状态会**临时或永久影响属性**：

| 状态 | 触发条件 | 效果 | 持续时间 | 解除方式 |
|------|---------|------|---------|---------|
| 愉悦 | 晴天出战/参战 | **全属性 +5%** | 当日有效 | 次日重置 |
| 正常 | 默认 | 无 | — | — |
| 感冒 | 雨天/雪天出战后，8%/6% 概率 | **全属性 -35%** | 5 天自然恢复 | ① 购买感冒药（金币）立即解除；② 等待 5 天 |
| 永久损伤 | 感冒自然恢复后，**2%~5% 概率**触发 | **全属性永久 -5%** | 永久 | 不可解除（但可通过升级/培养逐步弥补）|

**感冒机制详细**：

```
雨/雪天出战完成捕获或战斗
		│
		▼
   概率判定（雨8%/雪6%）
	  ├─ 未触发 → 正常
	  └─ 触发 → 进入【感冒】状态（属性-35%）
					│
			┌───────┴────────┐
			▼                ▼
	  使用感冒药           等待5天
	  （金币，立即解除）   （自然恢复）
			│                │
			▼                ▼
		 恢复正常        恢复判定
							  │
					┌─────────┴─────────┐
					▼                   ▼
			  95%~98% 概率        2%~5% 概率
			  完全恢复            永久损伤-5%
```

**设计意图**：
- 天气系统增加策略维度（雨天是否出战需权衡）
- 感冒药形成金币消耗出口，经济循环闭环
- 永久损伤概率极低（2%~5%），作为"不治疗"的风险代价，鼓励玩家主动用药
- 永久损伤幅度小（-5%），不影响普通玩家体验，但竞技向玩家会规避

### 5.5 等级与体力数值

**等级成长三要素**：
1. **体力上限随等级提升**：Lv.1 = 120 点 → Lv.10（满级）= 240 点（翻倍）
2. **升级即时奖励**：恢复满体力 + 金币回馈
3. **每日商店刷新加成**：等级越高，每日商店刷出高稀有道具的概率越高

| 等级 | 解锁条件（累计捕获数）| 体力上限 | 升级奖励 | 商店稀有道具刷新率 |
|------|---------------------|---------|---------|------------------|
| Lv.1 | 起始 | 120 | — | 基础 |
| Lv.2 | 10 只 | 134 | 恢复满体力 + 100 金币 | +2% |
| Lv.3 | 25 只 | 148 | 恢复满体力 + 150 金币 | +4% |
| Lv.4 | 45 只 | 162 | 恢复满体力 + 200 金币 | +6% |
| Lv.5 | 70 只 | 176 | 恢复满体力 + 300 金币 | +8% |
| Lv.6 | 100 只 | 190 | 恢复满体力 + 400 金币 | +10% |
| Lv.7 | 140 只 | 204 | 恢复满体力 + 500 金币 | +12% |
| Lv.8 | 190 只 | 218 | 恢复满体力 + 600 金币 | +14% |
| Lv.9 | 250 只 | 232 | 恢复满体力 + 700 金币 | +16% |
| Lv.10（满级）| 320 只 | 240 | 恢复满体力 + 1000 金币 + 限定称号 | +20% |

> 满体力 120 点（Lv.1）= 6 次捕获 / 派遣；满级 240 点 = 12 次。每小时恢复 10 点（每 6 分钟 +1），12 小时回满。体力双用途（捕获 / 淘金币）让玩家在体力富余时也有事可做。

### 5.6 亲密度与区域声望系统（新增）

> **注意**：聊天记录中提出的"多人扫描同一只动物可升级"因云端 VLM 仍无法可靠确认是否为同一生物（无唯一生物标识），故放弃"个体识别"路线，改为**区域声望**（玩家与区域绑定）+ **个体亲密度**（玩家与已捕获宠物）双轨设计。

#### 个体亲密度（Affinity）

玩家与已捕获宠物之间的羁绊值，通过日常互动提升。

**提升方式**：

| 行为 | 亲密度增量 | 说明 |
|------|-----------|------|
| 喂食 | +5/次 | 每日上限 3 次 |
| 抚摸（居家小游戏）| +8/次 | 每日上限 3 次 |
| 出战（PvP/PvE）| +3/次 | 每场战斗 |
| 合影（首次拍照）| +15 | 每只宠物仅一次 |
| 派遣任务 | +2/次 | 每次派遣归来 |

**等级与效果**：

| 亲密度等级 | 所需总值 | 效果 |
|-----------|---------|------|
| 陌生 | 0~29 | 无 |
| 熟悉 | 30~79 | 暴击率 +2% |
| 友好 | 80~149 | 暴击率 +4%，技能释放率 +2% |
| 亲密 | 150~249 | 暴击率 +6%，技能释放率 +4%，全属性 +3% |
| 挚友 | 250~399 | 暴击率 +8%，技能释放率 +6%，全属性 +5%，解锁特殊对话 |
| 灵魂伴侣 | 400+ | 暴击率 +10%，技能释放率 +8%，全属性 +8%，解锁专属皮肤/称号 |

**衰减机制**：连续 7 天无互动，亲密度每日 -5（最多降至该等级下限，不跨级跌落）。

**设计意图**：鼓励玩家持续关注自己的宠物，而非捕获后即弃；"灵魂伴侣"作为长线目标，提供情感锚点。

#### 区域声望（Regional Reputation）

玩家与特定地理区域的绑定关系，取代"个体识别"的不可行方案。

**核心概念**：以地级市为粒度，玩家在该城市的行为累积为"区域声望值"。

**声望提升**：

| 行为 | 声望增量 | 说明 |
|------|---------|------|
| 在该城市捕获动物 | +2/只 | 每日上限 20 点 |
| 在该城市战斗胜利 | +5/场 | PvP/PvE 均计 |
| 完成该城市派遣任务 | +3/次 | 居家任务绑定城市 |
| 参与区域 Boss 战 | +10/次 | 见 10.4 |

**声望等级**：

| 等级 | 声望范围 | 效果（仅在该城市生效）|
|------|---------|----------------------|
| 过客 | 0~49 | 无 |
| 熟人 | 50~149 | 捕获基础成功率 +2% |
| 常客 | 150~299 | 捕获基础成功率 +4%，战斗全属性 +3% |
| 地头蛇 | 300~499 | 捕获基础成功率 +6%，战斗全属性 +5%，派遣产出 +10% |
| 城市守护者 | 500+ | 捕获基础成功率 +8%，战斗全属性 +8%，派遣产出 +20%，解锁专属称号 |

**多城市独立**：每个城市的声望独立计算，玩家可在多个城市积累声望。

**设计意图**：将"同一只动物升级"的不可行概念转化为"玩家与城市的情感绑定"，鼓励旅行探索（跨城市游玩），同时为 LBS 核心体验提供正反馈。

#### 亲密度道具（金币消耗）

| 道具 | 效果 | 价格（金币）| 说明 |
|------|------|-----------|------|
| 高级零食 | 喂食时亲密度 +10（替代基础 +5）| 30 | 单次 |
| 爱心玩具 | 抚摸小游戏亲密度 +15（替代基础 +8）| 50 | 单次 |
| 纪念相框 | 合影亲密度 +25（替代基础 +15）| 80 | 单次 |
| 亲密度礼物盒 | 随机亲密度 +20~50 | 100 | 概率浮动 |

> 亲密度道具作为金币消耗新出口，与感冒药、体力药剂形成多元消耗矩阵，避免单一消耗路径导致经济通胀。

### 6.3 区域排行系统（新增）

**机制**：
- 按玩家 GPS 所在省份聚合"今日捕获总数"
- 每日 0:00 结算，次日生效

**奖励**：

| 排名 | 区域加成 | 金币奖励（该省全体玩家）|
|------|---------|----------------------|
| Top 1 | 全属性 +5%（当日生效）| 50 金币/人 |
| Top 2 | 全属性 +3%（当日生效）| 30 金币/人 |
| Top 3 | 全属性 +2%（当日生效）| 20 金币/人 |
| Top 4~10 | — | 10 金币/人 |

**设计意图**：
- 加成幅度小（+2%~5%），不破坏数值平衡，但竞技向玩家会追求
- 促进区域社交与拉新（"帮我们省冲榜"）
- 金币奖励形成稳定的软货币产出

**防刷**：
- 每日每省捕获数去重（同设备同日只计 1 次有效）
- 异常增长告警

### 6.5 战斗系统（巷弄冲突）详情

- **类型**：自动回合制（玩家战前排布阵容，战斗自动进行）
- **场景**：城市巷弄主题，呼应 LBS 真实场景
- **模式**：PvP 排位赛 + PvE 流浪动物挑战 + 周末联盟战
- **阵容**：3 只宠物 vs 3 只宠物，按 SPD 决定行动顺序
- **策略点**：元素克制、Class 配合（坦/输/辅/控）、出手顺序、**天气状态联动**（晴天宠物愉悦属性 +5%，雨天感冒属性 -35%，详见 3.5 / 5.4）
- **天气联动**：雨天水元素 +10% 伤害，火元素 -10%；晴天反之

| 风险类别 | 具体风险 | 应对策略 |

| **品种识别准确率** | 云端 VLM 误判导致稀有度争议 | 1) 各物种 Top 品种覆盖 90% 场景，长尾归为"混种"；2) 公开算法白皮书；3) 保留申诉通道 |
| **多物种检测** | 猫/鹅/狗混淆，鹅与鸭难区分 | 1) Prompt 持续迭代 + 误判样本反馈；2) 低置信度归为"未知物种"待二次确认；3) 持续收集误判样本迭代 |
| **防作弊能力** | 翻拍照片、模拟器、改包刷稀有度 | 1) 实时取景 + 时间戳校验；2) root/越狱检测；3) 服务端异常行为审计（同设备高频高稀有度告警）；4) 云端推理请求 ID 全链路留存 |
| **隐私合规** | 拍到人脸/车牌等敏感信息 + 照片经后端上传云端 | 1) 上传前客户端人脸模糊预处理；2) 照片经 Go 后端转发 VLM，后端即时销毁、不落盘不入训练；3) 首次授权明确告知照片上传与销毁机制；4) 通过个人信息保护法合规审计 |
| **网络依赖** | 弱网/断网导致无法发现、捕获 | 1) 弱网降级为"扫描式"单帧判定；2) 断网时仅保留图鉴浏览；3) 明确网络状态提示，避免误操作；4) 在线恢复自动续接 |
| **云端推理成本** | VLM/LLM 调用费用随 DAU 线性增长 | 1) 关键帧采样降低调用频次；2) 分级调用（检测走轻量接口，深度分析仅捕获后一次）；3) 结果缓存复用；4) Go 后端集中做单用户每日调用上限与成本告警 |
| **UI 卡顿** | CatchCat 滚动卡顿问题 | 1) 图鉴列表虚拟化渲染；2) 关键路径 Godot 原生 UI；3) 性能基线卡（中端机 30fps）纳入每版本验收 |
| **天气数据** | 一周固定恶劣天气累积感冒、玩家觉得被惩罚 | 1) 感冒概率调低（雨8%/雪6%）；2) 游戏内按地级市生成，无外部 API 依赖；3) 感冒药价格合理；4) 玩家可跨城市旅行切换天气 |
| **感冒机制争议** | 一周固定雨天/雪天累积感冒，玩家觉得惩罚重 | 1) 感冒概率调低（雨8%/雪6%）；2) 感冒药价格合理；3) 永久损伤概率极低（2%~5%）、幅度小（-5%），可通过升级培养弥补 |
| **多语言** | 后期接入成本高 | 1) 文案与代码分离，i18n 框架前置；2) 预留资源包按需下载 |
| **续航/发热** | 持续相机 + 上传导致发热 | 1) 关键帧采样自适应降低上传频率；2) 单次流程 ≤ 5min；3) 低电量模式降级为扫描式交互 |
| **内容枯竭** | 玩家后期无新鲜感 | 1) 季节限定动物；2) 区域限定（LBS 联动）；3) 联盟战、赛季制长线目标；4) 持续新增物种 |

## 十、玩法深化设计（丰富度提升）

> 基于玩法丰富度评估：核心循环中"捕获""战斗"两块单薄，"派遣""居家"单薄，且宠物养成/PVE/社交/长线目标四个方向完全缺失。本章针对这些做深化，提升可长期游玩的深度。

### 10.1 捕获小游戏深化（核心循环 · 解决单调）

**命中反馈梯度**（从二元"中/不中"升级为梯度）：

| 命中类型 | 判定 | 反馈 |
|---------|------|------|
| 完美命中 | 命中中心区域 | 捕获率 +15% + 额外金币 + 粒子特效 |
| 良好命中 | 命中目标 | 正常捕获 |
| 差点命中 | 擦边 | 30% 概率捕获 + 安抚动画 |
| 未命中 | 完全偏离 | 消耗投掷物，无捕获 |

**连击系统**：连续 3 次命中触发 buff（下次捕获 +10% + 双倍金币），未命中重置。

**动物行为变化**（让目标动起来）：

| 行为 | 表现 | 应对 |
|------|------|------|
| 警戒 | 抬头观察、移动加快 | 快速投掷 |
| 诱饵吸引 | 诱饵道具引至固定位置 | 配合诱饵 |
| 群体出现 | 多只同框，一掷可多中 | 预判落点 |
| 逃跑 | 转身离开，限时内须命中 | 倒计时压力 |

**环境交互**：障碍物（树枝/栏杆）弹开投掷物；风向影响抛物线；弹跳台/水面二次弹跳。

**季节变体**：冬季雪地投掷物滑动偏移；夏季水域弹跳反弹；秋季落叶遮挡视线。

### 10.2 战斗系统深化（核心循环 · 解决策略浅）

原"全自动回合制"改为**半自动 + 操作深度**：

- **自动普攻**（按 SPD 顺序）+ **能量条大招**：普攻/受击积累能量，满后手动释放大招（攒满即用 vs 等更优时机的策略选择）
- **站位系统**：前排承伤 70%（坦/控优选）、中排平衡、后排仅受 30%（输出优选）；回合间可调整站位（消耗 1 回合行动）
- **技能系统**：每只宠物装备 2 个主动技能（从技能池选），有冷却；同元素技能连续释放触发组合效果
- **战斗模式扩展**：PvP 段位赛 / PvE 关卡 / 联盟战（周末多人协作）/ 速通挑战（限时通关排名）

### 10.3 宠物养成系统（新增 · 连接收藏与战斗）

**培养升级**：宠物通过出战/训练获经验升级；每 10 级可"突破"（消耗同类碎片 + 金币），突破提升属性上限。

**进化系统**：满足条件（等级 + 亲密度 + 特定道具）触发形态进化，解锁新外观 + 新技能槽 + 属性跃升；部分宠物有多分支进化（玩家选路线）。

**亲密度系统**：抚摸/喂食/出战/合影提升；影响战斗暴击率、技能释放概率、进化条件；长期不出战/不互动会衰减（鼓励持续养成）。→ **详细数值与等级规则见 5.6 个体亲密度**。

**技能树**：每只宠物有专属技能树（3-4 分支），升级获技能点分配；同品种可走不同流派（增加多样性，避免"最优解"固化）。

### 10.4 PVE 内容（新增 · 单机深度）

**剧情章节**：玩家作为"动物图鉴员"探索各城市完成编撰任务；每章 5-10 关（对话/探索/战斗/Boss），解锁新区域/物种/道具。

**区域 Boss 战**：每城市 1-2 只"传奇动物"Boss（高稀有度 + 特殊机制），需 2-3 人组队；击败后概率捕获（极高稀有度）。

**挑战副本**：

| 类型 | 说明 | 奖励 |
|------|------|------|
| 每日挑战 | 随机 3 场战斗 | 金币 + 碎片 |
| 周挑战 | 强力 NPC 队伍 | 稀有道具 + 皮肤碎片 |
| 无尽塔 | 层层递进，记最高层 | 排行 + 限定称号 |
| 限时活动副本 | 节日主题 | 限定宠物/皮肤 |

### 10.5 社交玩法（新增 · 留存驱动）

**公会/社团**：创建或加入（上限 30 人）；每日协作公会任务；周末公会对抗赛；公会等级越高商店折扣/派遣加成越大。

**宠物交换**：好友间交换，受稀有度限制（同稀有度交换防刷）；每周 1 次冷却；交换后亲密度重置（防养成转移套利）。

**好友互访深化**：访问好友宠物园互动获双方收益；代好友照看宠物（离线时代为训练）；2 人协作捕获（合力提升稀有动物捕获率）。

**社交展示**：个人主页（图鉴/称号/成就）；图鉴分享生成精美卡片到社交平台。

### 10.6 长线目标（新增 · 生命周期）

**赛季制**：8 周一赛季，含赛季任务 + 段位 + 奖励；赛季通行证（免费线 + 付费线）；赛季结束按段位发奖，新赛季重置段位。

**限定活动**：

| 活动 | 频率 | 内容 |
|------|------|------|
| 季节限定动物 | 每季 | 仅当季可捕获限定物种 |
| 节日主题 | 节日 | 春节/万圣/圣诞主题 + 限定皮肤 |
| 城市巡礼 | 每月 | 指定城市限定动物出现率提升 |
| 联动活动 | 不定期 | 与动物救助机构/宠物品牌联动 |

**收藏家称号体系**：初级收藏家（50 只）→ 资深（150）→ 图鉴大师（300 + 全物种）→ 城市守护者（本市 100 只）→ 季节猎人（集齐四季限定）→ 传奇图鉴员（全图鉴）。

**图鉴深度任务线**：每条目有深度任务（某品种赢 10 场/培养满级/进化形态），完成解锁背景故事 + 奖励。

### 10.7 派遣任务化 + 居家玩法细化

**派遣 → 探索任务**（从被动产出改为主动玩法）：
- 简短叙事文本（宠物去哪、遇到什么）
- 产出多样化：金币 + 概率碎片/道具/经验
- 风险事件：宠物可能受伤需治疗、可能发现稀有线索触发特殊事件
- 每日 3 个任务可选（不同时长/收益/风险，玩家选择）

**居家玩法细化**：

| 玩法 | 说明 |
|------|------|
| 宠物训练室 | 小游戏（抚摸节奏/喂食精度/训练反应），提升亲密度与经验 |
| 宠物对战 | PvP/PvE 随时可进行（见 10.2）|
| 云端图鉴漫游 | 每日 3 次远程观察他城图鉴，获碎片 |
| 好友互访 | 见 10.5 |
| 宠物园装饰 | 收集装饰物布置宠物园（收集向玩法）|

> 玩法深化落地后，核心循环有了操作深度与变化，收藏有了养成纵深，并补齐 PVE/社交/长线目标三大缺失，从"骨架"提升到"可长期游玩"。
